План: пересобрать загрузку данных HSGuru с нуля

1. Очистить старый импорт
- Удалить текущую логику парсинга `scrape-hsguru`, которая сейчас слишком хрупко читает HTML.
- Очистить старые строки из таблиц данных матчапов и архетипов, чтобы сайт не показывал смешанные/старые результаты.
- Оставить сами таблицы и публичное чтение, потому что они уже подходят для отображения матчап-таблицы и турнирного стратега.

2. Написать новый стабильный парсер HSGuru
- Загружать данные напрямую с:
  - `https://www.hsguru.com/matchups?min_archetype_sample=1&min_matchup_sample=1&rank=all`
  - `rank=legend`
  - `rank=top_legend`, сохраняя как `top_1k` внутри сайта.
- Не привязываться к старому `patch_35.0.3` как обязательному параметру: брать текущий период HSGuru по умолчанию или определять активный период из страницы.
- Парсить таблицу по реальной структуре HSGuru:
  - заголовки колонок `opponent_*` дают список архетипов и класс;
  - строка популярности в `thead` дает `popularity`;
  - первый `td` каждой строки дает общий винрейт и `total_games` из `aria-label` вида `Deck - 895909 games`;
  - matchup-ячейки дают винрейт и количество игр из `aria-label` вида `A versus B - 123343 games`.
- Сохранять winrate/popularity округленными до 1 знака, total_games и estimated_games числом.

3. Сделать импорт атомарным
- Не удалять рабочие данные до успешного парсинга.
- Сначала собрать все данные по выбранным рангам в памяти.
- Если хотя бы основной ранг `all` не распарсился — импорт не заменяет базу и возвращает ошибку.
- Если парсинг успешен — удалить старые данные для текущей даты/периода и вставить новые батчами.
- Вернуть в админку подробный итог: сколько архетипов, матчапов, сколько ячеек с количеством игр, какой период и дата.

4. Обновить выдачу данных на сайт
- Упростить `get-matchups`: отдавать последнюю успешную загрузку по выбранному рангу без жесткой зависимости от старого patch-параметра.
- Оставить пагинацию, чтобы не упираться в лимит 1000 строк.
- Добавить диагностические поля в ответ, чтобы на странице было понятно: данные из базы есть, какой ранг, период, дата, сколько строк.

5. Привести UI к новым данным
- В `MatchupTable` показывать корректные `total_games` и `estimated_games`, чтобы фильтры Min Archetype Games / Min Matchup Games реально работали.
- В `TournamentStrategist` использовать те же данные, без fallback на старую статическую матрицу, если база успешно загрузилась.
- Если база пуста или импорт не выполнен — показывать понятное сообщение вместо пустой таблицы/кривых цифр.
- На главной странице оставить центральный переход к таблице матчапов, который уже добавлен.

6. Проверка после внедрения
- Развернуть обновленные backend-функции.
- Запустить синхронизацию через функцию.
- Проверить базу: количество строк по `all`, `legend`, `top_1k`, наличие `total_games` и `estimated_games`.
- Проверить `get-matchups` напрямую: должны приходить архетипы и матчапы.
- Проверить в интерфейсе `/matchups` и `/tournament`, что данные отображаются и фильтры работают.

Технические детали
- Секрет для загрузки уже настроен через коннектор, новый ключ не нужен.
- Структуру таблиц менять, скорее всего, не понадобится; потребуется только операция очистки/перезагрузки данных.
- Основная причина текущей проблемы: старый парсер не достает `total_games`/`estimated_games`, жестко привязан к patch-периоду и недостаточно надежно читает HTML HSGuru. Из-за этого фильтры игр фактически не работают, а часть данных выглядит некорректной.
